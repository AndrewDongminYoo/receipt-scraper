# receipt-scraper overview

- Purpose: React Native CLI practice project to rebuild hands-on React Native muscle memory before starting a new job.
- Product target: a minimal receipt rewards mini-clone with receipt upload, receipt list, survey, and reward result flow.
- Phase model: repo uses Day 0 through Day 5 labels. Day 0 is environment recovery; Day 1-5 are feature delivery.
- Current state: React Native 0.84.1 app runs on iOS and Android; App.tsx currently renders a minimal placeholder screen; src/ exists with scaffolded placeholder directories but real feature implementation has not started.
- Constraints: this is a working prototype, not a production app or architecture playground. Keep structure and state management intentionally minimal.
- Main references: BLUEPRINT.md for the sprint plan, AGENTS.md for repository rules, CLAUDE.md for assistant-facing context.
